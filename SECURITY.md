# Security Policy

## 报告漏洞

如果你发现安全漏洞，请**不要**公开 Issue。

请发送邮件到项目维护者，我们将尽快响应。

## 安全实践

本项目关注以下安全领域：

- **认证令牌**：使用 httpOnly cookie 存储会话令牌
- **密码哈希**：scrypt + 随机盐
- **CSP 策略**：动态 nonce 注入防止 XSS
- **API 限流**：内存限流，每 profile 每端点 10 req/min

## 已知限制

- 当前为个人学习+开源展示项目，不处理金融/医疗等敏感数据
- PIN 码当前为 4 位数字，公开部署前需扩展为 6+ 位字母数字
- SQLite 适合单机部署，高并发场景需迁移至 PostgreSQL
