# 玄学互动 Web 应用 — 设计文档

## 概述

一个多人可用的玄学互动 Web 应用。用户通过简单的昵称+PIN 登录，创建个人档案后可使用多种玄学工具进行命理推算，结果数据严谨、准确。

## 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | Nuxt.js 3 (Vue 3 全栈) |
| 数据库 | SQLite |
| 前端样式 | TailwindCSS + 组件库（具体由 frontend-design 决定） |
| 玄学引擎 | 纯 TypeScript，前端直接计算 |
| 部署 | 阿里云 + Nginx |

## 核心设计决策

- **轻量登录**：昵称 + 4位 PIN，小范围朋友使用，PIN 明文存储可接受
- **session 持久化**：登录后生成 token，浏览器存储，刷新不丢失
- **档案驱动**：创建档案后，所有工具自动读取档案信息
- **工具先行**：首页直接展示工具列表，设计直观
- **前端计算**：玄学排盘全部在前端完成，后端只做数据持久化
- **阶段落地**：从简单到复杂，每个工具独立可交付

## 数据模型

### profiles（用户档案）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer (PK, 自增) | 自动生成 |
| nickname | text (唯一) | 用户昵称，登录标识 |
| pin | text (4位) | 登录 PIN 码 |
| birth_date | text (YYYY-MM-DD, 可选) | 出生日期 |
| birth_calendar | text (solar/lunar, 可选) | 出生日期历法（阳历/农历） |
| birth_hour | integer (0-23, 可选) | 出生小时 |
| birth_minute | integer (0-59, 可选) | 出生分钟 |
| gender | text (男/女, 可选) | 性别 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

### sessions（登录会话）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer (PK, 自增) | 自动生成 |
| profile_id | integer (FK → profiles.id) | 关联档案 |
| token | text (唯一) | 登录令牌 |
| created_at | datetime | 创建时间 |

- 登录/注册时服务端生成 token，存入 `sessions` 表
- 同时返回给客户端，存入 `localStorage`
- 后续 API 请求通过 `Authorization: Bearer <token>` 头携带
- 写入类 API 验证 token 确保所有权
- 退出时调用 `DELETE /api/auth/logout` 删除会话记录

### divination_results（测算记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer (PK) | 自动生成 |
| profile_id | integer (FK → profiles.id) | 关联档案 |
| type | text | 工具类型 (shengxiao / constellation / bazi / yijing / ziwei) |
| input_data | json | 输入参数 |
| result_data | json | 计算结果 |
| created_at | datetime | 测算时间 |

### 首页设计

首页参考通用产品首页设计原则：
- **顶部**：品牌 Logo + 当前档案昵称（右上角，点击展开下拉菜单：编辑档案/退出）
- **欢迎区**：如已登录，显示"你好，{昵称}" + 简短副标题
- **工具网格**：卡片式布局，每个工具一张卡片（图标 + 名称 + 简介），未上线的工具显示"即将上线"灰态
- **首次访问**：无档案时自动跳转登录页
- **布局**：响应式，移动端单列，平板两列，桌面三列

| 路由 | 页面 | 说明 |

| `/` | 首页 | 见下方首页设计说明 |

## 功能规划

### 阶段 1：基础系统
- 用户登录（昵称 + 4位 PIN）
- 档案管理（创建、编辑）
- 首页工具网格 + 导航
- 首页默认展示当前登录档案名

### 阶段 2：生肖 + 星座（验证链路）
- 生肖计算（标准生肖算法）
- 生肖五行属性、性格特征（标准参考资料）
- 十二星座划分、性格特征
- 此阶段纯查表，确保每条数据有据可查

### 阶段 3：八字排盘（核心）
- 节气数据（万年历）
- 年柱推算（立春为界）
- 月柱推算（五虎遁）
- 日柱推算（日干支公式）
- 时柱推算（五鼠遁）
- 十神定位
- 五行生克分析
- 日主强弱判断

### 阶段 4：八字增强（深度）
- 神煞系统（权威全量，覆盖四柱 + 大运）
- 流年详批（前后各5年，规则驱动解读）
- 测算记录保存（结果持久化 + 历史回看）

### 阶段 5：六爻占卜（交互型）
- 数字起卦 / 手动摇卦动画
- 六十四卦系统
- 卦象解读

### 阶段 6：紫微斗数（最复杂）
- 十二宫排盘
- 星曜安放
- 命盘可视化

### 后续阶段（待规划）
- 合婚匹配（双方八字对比分析）
- 流月详批（流年之下细分到月）
- 星座/生肖深度增强

## 计算规则来源原则

1. 每种工具的计算规则在实现前，必须搜索行业标准资料确认
2. 数据（如节气、干支表、十神表、六十甲子等）使用权威来源
3. 不编造任何性格描述或命理判断——全部有实测依据
4. 实现阶段需要用户协助确认规则时，及时提问

## 架构原则

- 玄学计算引擎全部为 TypeScript 纯函数，与 UI 解耦
- 计算模块放在 `composables/` 或 `utils/`，可单独测试
- 不引入外部 API 依赖，所有计算离线可完成
- 前端负责展示，后端负责持久化
