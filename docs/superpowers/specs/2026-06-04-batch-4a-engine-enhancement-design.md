## 一、八字引擎增强

### 1.1 日主强弱加权评分

**现状：** `getDayMasterStrength(dayMasterWuxing, monthBranchIndex)` 仅按月支打分。

**目标：** 四柱 + 藏干加权评分，取代单一月支查表法。

**新函数签名：**
```typescript
export function getWeightedDayMasterStrength(
  dayMasterWuxing: string,
  pillars: BaZiPillar[],
): '强' | '偏强' | '中和' | '偏弱' | '弱'
```

**权重分配：**

| 因子 | 权重 | 说明 |
|------|------|------|
| 月支 | ×2.0 | 月令最强 |
| 日支 | ×1.5 | 日主之根，夫妻宫 |
| 月干 | ×1.0 | 月柱天干 |
| 年干、日干 | ×1.0 | — |
| 年支、时支 | ×1.0 | — |
| 时干 | ×1.0 | 晚年、子女 |
| 每个藏干 | ×0.5 | 地支中所藏人元 |

**评分规则：**
- 因子元素与日主相同 → +1
- 因子元素生助日主（生日主） → +0.5
- 因子元素克制日主（克日主） → -0.5
- 总加权分 ≥3 → 强, ≥1 → 偏强, ≥-1 → 中和, ≥-3 → 偏弱, 其他 → 弱

**保留向后兼容：** 旧 `getDayMasterStrength()` 保留但标记 `@deprecated`，新函数在 `calculateBaZi()` 内部替换调用。

### 1.2 调候分析

**目标：** 根据出生月加入气候调候维度，修正喜用神判断。

**新函数：**
```typescript
export function getSeasonalAdjustment(monthBranchIndex: number): string | null
```

**调候表（来源：子平真诠评注）：**

| 月支 | 气候特征 | 调候用神 | 说明 |
|------|---------|---------|------|
| 寅卯辰（春） | 木旺 | 金 | 木旺需金制，成材 |
| 巳午未（夏） | 火炎 | 水 | 火炎需水降，润局 |
| 申酉戌（秋） | 金锐 | 火 | 金锐需火炼，成器 |
| 亥子丑（冬） | 水寒 | 火 | 水寒需火暖，调候 |

**修改 `getFavorableElements()`：**
- 若调候用神已存在于喜用列表中 → 不变
- 若调候用神不在喜用列表中 → 追加到喜用列表首位
- 调候用神若恰好是忌神 → 说明"虽为忌神，但为调候所需"

### 1.3 纳音纳入五行统计

**现状：** `computeElementCounts()` 统计天干、地支、藏干的五行，但不含纳音。

**改动：** 在 `computeElementCounts()` 中增加纳音五行计数：
- 每柱纳音五行 ×1.0 权重
- 纳音五行通过 `getNayinWuxing()` 获取（从 `constants/hehun.ts` 抽取 `nayinWuxing` 逻辑成为共享函数至 `constants/bazi.ts`）

**提取共享函数至 `constants/bazi.ts`：**
```typescript
/** 六十甲子纳音五行映射 */
const NAYIN_WUXING_MAP: Record<string, string> = { /** ... 完整映射 */ }

/** 获取某柱干支的纳音五行 */
export function getNayinWuxing(stem: string, branch: string): string
```

### 1.4 解读增强

**现状：** `BaZiPillar` 只有 stem/branch/wuxing/tenGod 等结构化数据，无文字解读。

**改动：** `BaZiPillar` 新增 `interpretation?: string` 字段，在 `calculateBaZi()` 中通过规则模板生成。

**规则模板逻辑（以日柱外的其他三柱）：**

```typescript
function getPillarInterpretation(
  stem: string, branch: string, tenGod: string, pillarName: string
): string
```

**模板分类：**

| 维度 | 规则数 | 示例 |
|------|--------|------|
| 天干属性 | 10 | 甲→"甲木参天，栋梁之材，" / 乙→"乙木柔韧，顺势而为，" |
| 十神组合 | 10 | 正印→"得正印护身，" / 七杀→"七杀攻身，压力见动力，" |
| 地支藏干 | 12 | 子→"子水藏癸，" / 午→"午火藏丁己，" |
| 综合拼接 | ~30 | 如"甲木参天×正印×年柱" → "甲木参天，得正印护身，祖上积德之象。" |

**拼接优先级：**
1. 天干定性 + 十神性质 + 地支补充 → 完整句子
2. 若天干在地支有根（地支藏干含同五行）→ 附加"通根有力" / "根浅势弱"
3. 不同柱特殊含义：年柱（祖上/早年）、月柱（父母/青年）、时柱（子女/晚年）

**数据搜索要求：**
- 天干 10 个 → 查《三命通会》各天干赋文，提炼关键特质
- 十神 10 个 → 查子平法十神心性描述
- 地支 12 个 → 查地支藏干本性
- 全部来自权威术数文献，不编造

### 2.4 宫位差异化解读

**现状：** `PALACE_INTERPRETATIONS` 每个宫一句固定文本，不随入星星曜变化。

**目标：** 在 `useZiwei.ts` 的 `getDetailedPalaceView()` 中，按"宫位基础 + 主星解读 + 辅星提及 + 组合提及"模板动态生成。

**模板结构：**
```
"{宫位基本含义}。{主星解读}{辅星提及}{组合提及}"
```

- **宫位基本含义** — 来自 `PALACE_INTERPRETATIONS`（不变）
- **主星解读** — 来自 `STAR_INTERPRETATIONS`（如有主星），拼接："{星名}{解读}在{宫位}，"
- **辅星提及** — 如辅星列表非空，拼接："辅以{星名}，" + 解读
- **组合提及** — 查 `COMBINATION_INTERPRETATIONS`，如有匹配则加入

**示例：**
- 命宫天梁 + 辅星天魁 → "命宫代表一个人的先天禀赋。天梁入命，慈爱有长者风范。辅以天魁，贵人运强，逢凶化吉。"
- 夫妻宫天机 + 辅星天钺 + 铃星 → "夫妻宫代表婚姻状况。天机入夫妻宫，配偶聪慧但变化多。辅以天钺，得配偶助力。铃星暗藏，需防口角摩擦。"

**特殊处理：** 空宫（无主星）→ 对宫借星处理 + "借对宫{星名}照入"说明。
